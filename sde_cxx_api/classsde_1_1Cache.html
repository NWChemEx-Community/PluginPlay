) const

Returns a shared_ptr to the value held in an SDEAny instance stored in the result map. This function is used when the desired value is an intermediate result computed by a submodule, for which the associated input key is unknown. The result is instead specified by providing the hash value of the parent module, and the hash value of the submodule state (i.e. the return value of ModuleBase::Memoize() with no arguments).

This function currently has two major limitations: 1) It will only search to a nested call depth of one (Module->Submodule). 2) Will always return the last result computed by the submodule. No option to change this behavior.
