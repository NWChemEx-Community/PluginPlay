Detailed Description

template<typename T>
class pluginplay::detail_::AnyInputWrapper< T >

Implements input-specific methods of the any class.

Template Parameters
TThe type of the wrapped object. Expected to be by value, by const value, or by const reference.

This is the most derived class of the AnyInput implementation. It is responsible for implementing the input specific functionality. Functionality common to both AnyInput and AnyResult is implemented in the AnyFieldWrapper base class.

template<typename U , typename = disable_if_any_input_wrapper_t<std::decay_t<U>>>
pluginplay::detail_::AnyInputWrapper< T >::AnyInputWrapper ( U &&  value2wrap)

Makes an AnyInputWrapper by wrapping the provided value.

Template Parameters
UType of the value we are wrapping. U must be implicitly convertible to T.
<anonymous>Used to disable this function via SFINAE when U is an instantiation of an input wrapper.

This ctor wraps the provided value in a newly created AnyInputWrapper. Exactly how this is done depends on how value2wrap is passed and the exact type of T.

  • When T is a value or const value the resulting AnyInputWrapper instance will own the wrapped value. This means the wrapped value will be a copy of value2wrap if value2wrap was passed by reference and it will take ownership of value2wrap if it was passed by value or rvalue.
  • When T is a const reference AnyInputWrapper will only alias value2wrap and the user is responsible for keeping value2wrap alive for the lifetime of the AnyInputWrapper.

N.B. if U is not an implicityly convertible to type T you will get a compiler error saying there is no valid std::any ctor for U.

Creates a new AnyInputWrapper by deep copying another AnyInputWrapper instance.

This ctor creates a new AnyInputWrapper by deep copying from another instance. Exactly what deep copy means depends on the type being wrapped. When T is a value (or const value) it will be whatever the type T defines a deep copy as. When T is a read-only reference the read-only reference will be deep copied; in turn, the resulting read- only reference will still alias the original value.
