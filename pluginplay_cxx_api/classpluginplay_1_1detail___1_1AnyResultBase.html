Detailed Description

Extends the AnyFieldBase API to result-specific functions.

In PluginPlay results are owned by AnyResult instances. The results can only be accessed by value and/or read-only reference. This class implements the retrieval of the wrapped object while enforcing these restrictions. Additionally, the AnyResultBase class defines functions which are needed to implement the AnyResult class, but not the AnyInput class.

N.B. Since PluginPlay always creates either an AnyInput or an AnyResult we have opted to put the clone method in the AnyResultBase/AnyInputBase classes to avoid AnyInput and AnyResult having to downcast from an AnyFieldBase object.

template<typename T >
T pluginplay::detail_::AnyResultBase::cast

Retrieves the wrapped object as an instance of type T.

Results are owned by the AnyResult and can only be interacted with in a read-only manner. This function allows you to get the wrapped object back as an object of type T as long as T is consistent with the read-only restrictions and T only differs from the actual type of the stored object up to cv-qualifiers and a reference.

N.B. As the ctors/assignment operators are setup, it is not possible for an AnyResultBase instance to not hold a value. If an AnyResult doesn't hold a value it's because it has a null AnyResultBase pointer.

template<typename T >
bool pluginplay::detail_::AnyResultBase::is_convertible
noexcept

Determines if the wrapped type can be converted to type T.

The wrapped object is convertible to a type T if:

  • T is consistent with the read-only nature of the held data, and
  • the type of the wrapped object differs from type T by at most a reference and/or cv qualifiers.
Returns
True if the wrapped object can be retrieved as an object of type @T and false otherwise.
